更新日誌

v2.21.61（2026-05-25）

OpenClaw 連線：WebSocket 握手協定升級為 v3–v4，修正新版 Gateway 回報 protocol mismatch（expected=4）而無法 Connect / 載入 Session。Gateway URL 支援 http(s):// 與 ws(s)://，http(s) 會自動轉為 ws(s)。

v2.21.60（2026-05-25）

產品名稱：Extension 顯示名稱由 Momo AI Bud 更新為 Hii~ Momo: AI Assist。同步更新 manifest、側邊欄 / 設定頁標題、多語系文件標題、浮球 tooltip、web search 標記與 README。

v2.21.59（2026-05-25）

引用頁面圖示：修正 assistant action bar 只要整個 session 曾有引用頁面，就會在後續 AI 回覆也顯示 reference page icon 的問題。現在只有該 assistant 回覆前一條 user message 本身帶有 Page Referenced 時才顯示圖示，並只開啟該輪綁定的頁面內容。

v2.21.58（2026-05-25）

引用頁面 token 成本：修正引用頁面內容綁定到 user message 後，後續多輪對話仍可能因歷史訊息一起送出而重複注入整頁內容的問題。現在只有最新一條 user message 本身帶有 Page Referenced 時，才會把頁面內容注入 API payload；後續追問不再重送整份頁面。

v2.21.57（2026-05-25）

Smart Capture：引用頁面擷取模式新增 Smart Capture (Recommended) 與 Visible Text。Smart Capture 會依頁型自動選擇較省 token 的策略：文章 / 文件頁走 Markdown 智能提取，產品頁、定價頁、dashboard、settings、表格 / 卡片 / 按鈕密集頁走可見文字擷取。Visible Text 只收集實際可見的標題、段落、列表、表格、按鈕與價格 / plan / feature 類區塊並去重。

v2.21.56（2026-05-25）

Markdown 智能提取：修正產品頁 / SPA 頁面可能被 Mozilla Readability 誤判，導致引用 URL 正確但內容抓到隱藏模板或其他文章片段的問題。擷取結果現在會與目前頁面的 title / H1 關鍵字比對；若不一致，會改用 fallback DOM 與實際可見文字 document.body.innerText 作保底。

v2.21.55（2026-05-25）

引用頁面送出穩定性：修正畫面已顯示 Page Referenced，但部分 OpenAI-compatible provider（例如 Qwen 類模型）回覆仍像沒有看到頁面內容的問題。引用頁面內容現在會直接綁定到當次 user message 的 API payload，不再依賴對話中途插入的 system message；聯網搜尋判斷也改用原始使用者輸入，避免整份頁面內容被誤當成搜尋 query。

v2.21.54（2026-05-24）

Hermes / OpenClaw 連線狀態：設定頁刷新或切換外觀主題後，Hermes / OpenClaw 的 Connect 狀態與模型開關會正確恢復，不再要求重新 Connect。連線狀態依 Base URL、API key、OpenClaw session key 比對；手動 Disconnect 仍會清除。側邊欄亦會同步設定頁的模型選擇。

v2.21.53（2026-05-24）

Agent 編輯捲動：Hermes / OpenClaw 編輯舊訊息後重新發送時，對話會穩定捲至最新 user / assistant 訊息，不再出現先跳到底再彈回的問題。

v2.21.52（2026-05-24）

Agent 編輯行為：在 Hermes 或 OpenClaw 下編輯 user message 時，Momo 會將編輯後內容作為新訊息追加並重新送給 agent，不會截斷本地對話歷史（因遠端 session 無法由 Momo 假裝回退）。一般 OpenAI-compatible provider 仍維持原本「替換並截斷後續訊息」的行為。

v2.21.51（2026-05-24）

AI 回覆圖片去重：修正 Hermes / ComfyUI 回傳 MEDIA: 與 query 內裸圖片 URL 被當成兩張圖重複顯示的問題；MEDIA: 行只取完整 URL，ComfyUI /view 連結會正規化去重。

v2.21.50（2026-05-24）

AI 回覆圖片：AI 回覆若包含 MEDIA:<url>、Markdown 圖片或 HTTP/HTTPS 圖片 URL，Momo 會自動下載並在對話中內嵌顯示，成功時存入本機 IndexedDB，不只保留外部連結。Hermes / OpenClaw 的 HTTP 圖片回覆（含 OpenClaw image / image_url content part）亦支援。詳見圖片上傳 — AI 回覆中的圖片。

v2.21.24（2026-05-07）

側邊欄輸入啟動：修正打開 Momo AI Bud sidepanel 後，初始化完成才清空輸入框，導致使用者剛輸入的文字可能被清掉、體感像需要等待才能打字的問題。輸入框 reset 現在會在載入模型、語言與歷史前先完成，開啟後可立即輸入。

v2.21.23（2026-05-05）

Hermes Setup Guide i18n：Hermes Setup Guide modal 改用 i18n key，補齊繁中、簡中、英文三套文案；.env 範例改為真正多行 code block，避免在 modal 內被壓成單行難以閱讀。README 與 AI Providers 文件亦補充 Hermes Agent 介紹與使用建議。

v2.21.22（2026-05-05）

Hermes Setup Guide：Hermes (beta) 現在和 OpenClaw (beta) 一樣，在 Connect row 右側顯示 Setup Guide 按鈕。Setup Guide modal 會依目前 provider 切換內容；Hermes 會顯示 .env 範例與重啟 Hermes gateway 的步驟，包含 API_SERVER_CORS_ORIGINS=* 說明與 API_SERVER_KEY=<your-secret-key> 佔位。

v2.21.21（2026-05-05）

Hermes .env 說明：設定頁與 AI Providers 文件補充 Hermes Agent .env 範例，說明需像 OpenClaw gateway 一樣修改 API_SERVER_ENABLED、API_SERVER_HOST、API_SERVER_PORT、API_SERVER_KEY 與 API_SERVER_CORS_ORIGINS。範例 key 改用 <your-secret-key> 佔位，並標明 API_SERVER_CORS_ORIGINS=* 是 Chrome extension / 遠端連線最簡單可用設定。

v2.21.20（2026-05-05）

Hermes Chat 403：Hermes chat request 改由 background proxy_fetch 代發，避免 sidepanel 直接 fetch 時與 curl 不一致的 browser Origin / CORS / preflight 差異造成 HTTP 403。Hermes /v1/chat/completions 也改用最小 OpenAI-compatible payload：model: "hermes-agent"、messages、stream: false。

v2.21.19（2026-05-05）

Hermes 連線與切換：Hermes 發生 HTTP 401/403 或 browser fetch 失敗時，錯誤訊息會直接顯示目前 Chrome extension origin，方便在 Hermes host 設定 API_SERVER_CORS_ORIGINS=chrome-extension://<extension-id> 後重啟 gateway。從其他模型切換到 Hermes 時會自動開新對話並清掉目前畫面的舊上下文；送出 API 前也會過濾本地 system message，確保 agent provider 不使用自訂 system prompt。

v2.21.18（2026-05-05）

OpenClaw 回覆穩定性：修正發送訊息後可能先重覆上一輪 assistant 回覆的問題。OpenClaw chat.send 改回 deliver: true，並用本次 runId 與發送前 history snapshot 過濾舊事件／舊 history，後備輪詢只接受本次新增訊息之後的新 assistant 回覆。

v2.21.17（2026-05-05）
